# Web Calibration Guide

Use this guide when distilling entries where current external practice, official docs, known patterns, or industry terminology can sharpen the insight.

## Search Strategy

Search for the core technical concept, not the entry title.

Good query:

```text
Kafka consumer rebalance backpressure pattern
```

Weak query:

```text
2026-05-22_fix_kafka_poll_membership_backpressure
```

Prefer primary and authoritative sources:

- Official documentation.
- Maintainer docs or design notes.
- Standards and specifications.
- Well-known engineering writeups when official docs are insufficient.

## Calibration Questions

Use external sources to answer:

1. Does the external consensus support the entry's claim?
2. Is there an established name for the pattern?
3. Are there known variants or boundary cases?
4. Are there concrete recommended parameters, defaults, or version constraints?
5. Does the original insight need correction?

## Output Shape


Do not paste long quotes. Paraphrase and link sources.
