# 技术栈认知

> 对具体技术栈的深度理解。超越文档的实战洞察。

## 核心关注

- LLM 应用（tool use、prompt 设计、模型选择）
- 消息队列（Kafka 消费模式、rebalance 行为）
- 数据库（事务模型、索引策略、分库分表、迁移实践）
- 前端（CSS 布局、权限控制、图表库）
- 安全（密码哈希、Token 管理、防爆破）
- 工具链（Git 状态恢复、部署运维）

## 已沉淀

### 数据库与存储
- **Redis ZSet 时序存储** — < 1万点/天时不需要专业时序数据库
- **基因法：ID 生成时编码路由信息** — 让非分片键也能精准路由到分片；只解决点查，不解决范围查询
- **ES 和分库分表互补** — ES 解决"怎么查"，分库分表解决"怎么存"；写入瓶颈/DDL/连接数 ES 管不了

### 前端
- **CSS class 切换 > JS 条件渲染** — 简单权限控制场景用 hidden/is-collapsed class；三层权限粒度：CSS → JS 事件拦截 → 路由守卫

### 工具链
- **Git 恢复是四层状态收敛** — 远端引用、HEAD 对象、分支名、工作树；遇到落盘问题时先检查对象层是否已到目标提交
